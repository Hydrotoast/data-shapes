Abstract

This vocabulary defines terms used in SHACL, the W3C Shapes Constraint Language. This vocabulary is under development by the W3C Data Shapes Working Group. The status of each term in this vocabulary is assumed to be stable unless they are marked using Term-centric Semantic Web Vocabulary Annotations.

sh:ConstraintComponent

General Info
rdf:type:
rdfs:Class
rdfs:label:
Constraint component
rdfs:comment:
The class of constraint components such as sh:PatternConstraintComponent. Constraint components serve as containers to group together the properties used in a constraint, for example sh:pattern and sh:flags. These properties can be linked to the constraint component via sh:parameter. Constraint components can also have links to validators (e.g. in SPARQL) that can be discovered by implementations, using the following properties:
  • sh:propertyValidator points at the validator if used in the context of sh:property
  • sh:inversePropertyValidator points at the validator if used in the context of sh:inverseProperty
  • sh:nodeValidator points at the validator if used in the context of sh:constraint

This constraint component can be used to describe the intersection of multiple shapes. This constraint component can be applied to node constraints only. The parameter sh:and points at a list of sh:Shape instances. Valid nodes must validate successfully against all shapes in the list.

sh:ClassConstraintComponent

General Info
rdf:type:
sh:ConstraintComponent
rdfs:label:
Class constraint component
rdfs:comment:
This constraint component can be used to specify a class that valid nodes must be instances of. This constraint component can be applied to:
  • node constraints to apply to all nodes in scope
  • property constraints to apply to all values of the given property
  • inverse property constraints to apply to all inverse property values
The parameter sh:class points at the class. Valid nodes must have an rdf:type triple to that class or a subclass thereof.
