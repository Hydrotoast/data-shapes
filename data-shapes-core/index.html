What are Shapes and Constraints

A shape describes the triples coming into and out of the current node in an RDF graph. A shape has constraints attached to it which describe the current node's triples. Some of these constraints will be about a specific property only, these are called property constraints. A property constraint for a triple with the current node as a subject is attached to the shape via sh:property; a property constraint for a triple with the current node as a object is attached to the shape via sh:inverseProperty.

sh:hasValue

The property constraint sh:hasValue can be used to verify that the current node has a given RDF node among the values of the given predicate.

TEXTUAL DEFINITION
An sh:Error must be reported if there is no triple that has the current node as its subject, the sh:predicate as its predicate and sh:hasValue as its object.
SPARQL DEFINITION
ASK {
	FILTER NOT EXISTS { ?this ?predicate ?hasValue }
}
ex:HasValueExampleShape
	a sh:Shape ;
	sh:property [
		sh:predicate ex:property ;
		sh:hasValue ex:Green ;
	] .

ex:HasValueExampleValidResource
	ex:property ex:Green .
