This document describes the SHACL test suite.

Initial version

Introduction

The contents are inspired by the RDF test suite.

The tests are available in the folder tests which can also be downloaded as a simple file: tests.tar.gz, tests.zip.

There are four types of tests:

  • Positive syntax
  • Negative syntax
  • Valid
  • Not valid

Namespace prefixes

sh
http://www.w3.org/ns/shacl#
mf
http://www.w3.org/2001/sw/DataAccess/tests/test-manifest#

Positive Syntax

These tests contain example of SHACL schema files which can be parsed. They have type sh:PositiveSyntax

Negative syntax

These tests contain examples with bad syntax. They have type sh:NegativeSyntax and their structure is the same as the previous tests.

Match

They have type sh:Match.

NoMatch

These tests contain schemas and RDFs that can be parsed. They also refer to a IRI in the graph and to a label in the schema. However, the IRI does not match that Shape

These tests have type sh:NoMatch
